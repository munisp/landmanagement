## 📊 System Overview

The **Integrated Digital Land Registry & Property Title System (IDLR-PTS)** is a comprehensive, enterprise-grade platform for managing land parcels, property transactions, and title documents with blockchain verification, advanced security, and full compliance with Nigerian and international regulations.

### Key Statistics

- **85+ Major Features** across 13 categories
- **9 Languages** supported (English, French, Arabic, Hausa, Yoruba, Igbo, Pidgin, Swahili, Amharic)
- **Polyglot Architecture** (TypeScript, Go, Python)
- **Blockchain Integration** (Hyperledger Fabric)
- **Production Infrastructure** (Kubernetes + Istio + Prometheus)
- **Enterprise Security** (MFA + Audit Trail + GDPR Compliance)

## 🚀 Deployment Instructions

### Prerequisites
- Kubernetes cluster (1.28+)
- Helm 3.x
- kubectl configured
- Docker registry access

### Quick Start
```bash
# 1. Clone repository
git clone https://github.com/your-org/idlr-pts-platform.git
cd idlr-pts-platform

# 2. Install dependencies
pnpm install

# 3. Configure environment
cp .env.example .env
# Edit .env with your configuration

# 4. Run database migrations
pnpm db:push

# 5. Start development server
pnpm dev

# 6. Run tests
pnpm test

# 7. Build for production
pnpm build

# 8. Deploy to Kubernetes
cd /home/ubuntu/idlr-k8s
helm install idlr-web ./helm/idlr-web
```

### Production Deployment
```bash
# Deploy all services
kubectl apply -f /home/ubuntu/idlr-k8s/

# Verify deployment
kubectl get pods -n idlr

# Check service health
kubectl get svc -n idlr
```
